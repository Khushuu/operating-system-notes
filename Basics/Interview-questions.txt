Operating System: 

A software that manages computer hardware.
Examples: Windows(microsoft), macOS(Apple), linux(open-source)

Process:

- instance of a program in execution
- web browser and command prompt are process 
- each process allocated some time to use the processor and other resources like memory/disk

Process table

- keeps track of state of all processes
- keeps list of process, resource, current state of the process
- states: running, ready or waiting
- only one process can be in running state at a time
- waiting: for external event to occur such as user input or disk access
- running: waiting for permission to use the processor
- both implemented as queues - holding processes 
- an array of PCBs 

File Allocation table (FAT) 

- maintained by OS on a hard disk 
- maintains a linked list structure that maps files to the clusters(smallest unit of storage) they occupy on the disk.

Thread

- single sequence stream within a process 
- also called lightweight process
- imporves the application through parallelism 
- web browser - multiple tabs can be different threads 

Difference between Process and thread

- process is program under action - works independently
- threads are lightweight process using same address space
- thread has its own program counter(PC), register set and stack space 
- threads can communicate and synchronize 

Thrashing

- when a system spends more time swapping pages in and out of memory than executing actual processes
- performance of computer degrades
- system spends more time processing page faults than executing transactions
- as page fault increases more transactions need processing from paging device
CAUSES 
- occurs when system's physical memory is too small to hold the active process 
- too many processes running simultaneously - exceeds capacity of the system's memory
- if degree of multiprogramming increases 

Buffer 

- a memory area that stores data being transferred between two devices or between device and an application

Virtual Memory 

- uses disk space to extend the RAM
- process doesn't care if the memory is from RAM or disk 

Demand Paging 

- process of loading the page into memory on demand(whenever a page fault occurs)

Kernel 

- central and core component of an OS 
- manages operation of memory and CPU time 
- bridge b/w applications and data processing performed at hardware level 

Multi-programming 

- increases CPU utilization by organizing jobs 
- keeps multiple jobs in main memory 
- if one job uses IO other can use CPU 

Concurrency 

- a state in which process exists simultaneously with another process

Overlays 

- when a process is running it will not use the complete program at the same time but only some part of it 
- load only a part of the program code 
- once processing of that part is done load the next part and unload the previous 
- formally:  “The process of transferring a block of program code or other data into internal memory, replacing what is already stored”.

Fragmentation 

- Processes are stored and removed from memory, which makes free memory space
- the memory space that is too little to be utilized by different process is fragmentation 
- occurs during dynamic memory allotment framework 

Compaction 

- process of collecting fragments of available memory space into contiguous blocks 
- done by moving programs and data in a computer's memory disk

Swapping 

- memory/process management technique used by OS to increase the utilization of the processor
- moves some blocked process from the main memory to the secondary memory 
- allows more processes to run that can fit into memory at one time 

Direct Access Method 

- Direct Memory Access(DMA) is a method that allows I/O device to send or receive data directly from main memory
- DMA bypasses CPU to speed up memory operations
- the way data is accessed on storage devices like hard drives, SSDs, and optical discs where each block of data is identified by a unique address
- This allows the system to retrieve or store data directly by specifying the address, without having to read through other data sequentially.
- Data can be accessed in any order, making it non-sequential.
- No need to traverse data blocks sequentially as in the sequential access method.

Cycle Stealing 

- accessing computer memory(RAM) or bus without interfering with or by bypassing CPU
- similar to Direct memory access that allows I/O controllers to read or write RAM without CPU intervention 

Multitasking 

- more than 1 task executed at the same time 
- multiple task also known as processes share common processing resource such as CPU 

Premptive Multitasking

- allows computer programs to share OS and hardware resources 
- divides operating and computing time between processes 
- switches resources b/w different processes using predefined criteria

Caching 

- Cache is smaller and faster memory 
- stores copy of data that is frequently used 
- Caching reduces average time to access data from main memory 

Associative Memory 

- memory unit accessed by content 
- reduces time required to find the item in the memory 
- data is accessed by its content 
- used where search time is very short 

Spooling (??)

- refers to keeping jobs in a buffer or on disk where a device can access them when it is ready 
- useful because devices access data at different rates

Assembler functionality 

- translates assembly language program to machine code 

Interrupts 

- signals emmitted by hardware of software when a process or event needs immediate attention 
- interrupts current working process 
- Interrupt Service Routing(ISR) - bus control lines dedicated for interrupts in I/O devices 

Trap 

- a software interrupt, usually the result of an error condition 
- it is non-maskable interrupt and has the highest priority 

Zombie process 

- a process that has finished the execution but still has an entry in the process table
- its exit status is still waiting to be collected by the parent proces
- the parent process "reaps" the zombie process by calling system functions like wait() or waitpid() to collect the child's exit status

Orphan processes 

- a process whose parent process no more exist i.e either finished or terminated 
- parent process did not wait for child process to terminate 

Starvation 

- resource management problem 
- process does not get the resources it needs for a long time
- happens because resources are being allocated to other processes 

Aging 

- technique to avoid starvation in a scheduling system 
- adds an aging factor to priority of each request 
- aging factor increases priority of each request 
- ensures that a request will eventually be the highest priority 

Context Switching 

- Switches CPU to another process 
- saves the state of the old process in Process Control Block (PCB)
- so that it can be resumed from the same part it was left
- and loads the saved state for the new process 

Process Control Block (PCB)

- tracks the process's execution status 
- contains info about the process such as registers, quantums, priority etc. 

Safe State 

- if there exists at least one temporal order in which all processes can be run to completion without resulting in a deadlock

Dispatch Latency 

- amount of time it takes for a system to respond to a request for a process to begin operation 

Critical Section

- code segment accessed by more than 1 process 

Bounded waiting 

- ensures that every process gets a fair chance to execute its critical section within a bounded number of attempts made by other processes.
- This guarantees that no process will suffer from indefinite postponement or starvation.
- After a process requests entry into its critical section, there must be an upper limit on how many other processes are allowed to enter before it does.
- A process will wait for its turn, but it is guaranteed to get access after a bounded number of attempts by the other process.

Peterson's Approach 

- concurrent programming algorithm 
- used to synchronize 2 processes that maintain mutual exclusion for the shared resource 
- uses two variabls - bool array[2] flag of size 2 and int turn (variable)
